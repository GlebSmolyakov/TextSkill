# TextSkill

Стоп-лист AI-штампов для русского текста, методика аудита и восемь скиллов Claude Code. В корне два файла: `writing-rules.md` с паттернами и правилами живого письма, `audit.md` с методикой проверки. Скиллы в `.claude/skills/` берут ядро и добавляют правила своего жанра: write-post знает лимиты Telegram и типы постов, write-docs переключает регистр на нейтральный, audit-text прогоняет текст по чек-листу и ставит балл 1–10.

Правила работают тремя слоями:

- **Ядро** (`writing-rules.md`): запрещённая лексика, синтаксические паттерны, ритм, типографика, плюс позитивные правила живого письма.
- **Формат**: каждый скилл добавляет правила своего жанра поверх ядра.
- **Голос** (`style-profile.local.md`, опционально): скилл `style-profile` собирает твои образцы и кладёт профиль в корень. Дальше скиллы пишут в твоём регистре, не в усреднённом.

## Структура

```
writing-rules.md                # ядро универсальных правил
audit.md                        # методика аудита: шкала и формат (паттерны берёт из ядра)
style-profile.local.md          # личный голос автора, локальный, не в гите (опционально)
.claude/skills/
├── write-post/SKILL.md         # пост для Telegram
├── write-article/SKILL.md      # статья / лонгрид
├── write-readme/SKILL.md       # README.md для GitHub
├── write-docs/SKILL.md         # техническая документация
├── audit-text/SKILL.md         # оценка человечности (детектор AI)
├── audit-docs/SKILL.md         # редакторский аудит документации, 1–10
├── rewrite-text/SKILL.md       # переписать, убрав AI-штампы
└── style-profile/SKILL.md      # собрать профиль авторского голоса
```

## Установка

Вариант 1. Клонировать репо и работать прямо в нём:

```bash
git clone git@github.com:GlebSmolyakov/TextSkill.git
cd TextSkill
claude
```

Вариант 2. Скопировать ядро и скиллы в свой проект:

```bash
cp -r TextSkill/{writing-rules.md,audit.md,.claude/skills} /путь/к/твоему/проекту/
```

Скиллы подключаются сами, как только Claude видит папку `.claude/skills/` в текущем проекте.

## Скиллы

| Скилл | Что делает |
|---|---|
| `write-post` | пост для Telegram: лимиты, шесть типов постов, маркировка рекламы по ФЗ-38, стоп-лист кликбейта |
| `write-article` | статья или лонгрид: открытие, тезис, ровная глубина через весь текст; для туториалов есть блок про работу с кодом |
| `write-readme` | README под GitHub: правила по типу проекта, шпаргалка по разметке, стоп-лист бэдж-супа и маркетинговых подвалов |
| `write-docs` | техническая документация: нейтральный регистр (override ядра), факты важнее стиля |
| `audit-text` | детектор AI: балл 1–10 и список паттернов по тяжести |
| `audit-docs` | редакторский аудит документации: 8 шагов, упор на фактическую точность, балл 1–10 |
| `rewrite-text` | переписать готовый текст: пять слоёв (лексика → синтаксис → ритм → структура → позиция), на выходе одна версия |
| `style-profile` | собрать твой голос в `style-profile.local.md` |

Что не влезло в таблицу:

- `audit-text` и `rewrite-text` под конкретный текст подтягивают форматный скилл (пост → `write-post`, статья → `write-article` и так далее). Так аудит ловит и запреты ядра, и нарушения формата.
- `write-docs` работает в инвертированном регистре: в доках нейтральный тон вместо позиции, обязательный вывод вместо открытого финала, числа до десяти словами. `audit-docs` за это не штрафует, для документации это норма. Профиль голоса на доки тоже не распространяется.
- `style-profile` ничего не пишет и не проверяет. Задаёт пару вопросов, разбирает твои образцы и кладёт `style-profile.local.md` в корень. Файл локальный, в гит не идёт; у каждого автора свой.

## Использование

В Claude Code открой проект и напиши, что нужно. Скилл подхватится по описанию:

- «напиши пост про провальную рекламную кампанию» → `write-post` спросит канал, тон и тип поста, выдаст пост по правилам Telegram.
- «опиши метод crm.item.add» → `write-docs` пишет справочную страницу в нейтральном доковом регистре.
- «проверь этот текст» + текст → `audit-text` гонит по чек-листу, возвращает балл 1–10 и паттерны по тяжести.
- «проверь эту документацию» + текст → `audit-docs` идёт по 8 шагам с упором на факты.
- «перепиши, убери AI-паттерны» + текст → `rewrite-text` спросит глубину и пройдёт по пяти слоям.
- «настрой мой стиль» / «пиши как я» → `style-profile` соберёт `style-profile.local.md`, и дальше скиллы живого текста пишут в твоём голосе.

## Если ты не в Claude Code

Промты для обычного чата.

**Пост для Telegram**
```
Используй правила из writing-rules.md как базу.
Дополнительно: .claude/skills/write-post/SKILL.md.

Тема: [тема]
Канал: [о чём канал, кто читает]
Тон: [разговорный / деловой / личный]
Тип поста: [личный / новостной / экспертный / рекламный / опрос / анонс]
```

**Статья**
```
Используй правила из writing-rules.md как базу.
Дополнительно: .claude/skills/write-article/SKILL.md.

Тема: [тема]
Тезис: [главная мысль]
Аудитория: [кто читает, что уже знает]
Объём: [примерно слов]
Тип: [эссе / аналитика / туториал / обзор / технические «кишочки» / DIY]
```

**Документация**
```
Используй writing-rules.md как базу.
Дополнительно: .claude/skills/write-docs/SKILL.md (сначала примени блок override регистра).
Если есть локальная редполитика репозитория с продуктовыми терминами, примени поверх.

Тип страницы: [справочная статья / описание метода или события / страница параметра / обзорная]
Тема: [что описываем]
```

**Переписать**
```
Используй writing-rules.md.
Если текст пост, добавь .claude/skills/write-post/SKILL.md.
Если статья, добавь .claude/skills/write-article/SKILL.md.
Если документация, добавь .claude/skills/write-docs/SKILL.md (его override отменяет правила ядра про позицию, финал, союзы, числа).

Перепиши: убери AI-паттерны, смысл и структуру сохрани.
Глубина: [лёгкая / средняя / глубокая]

Текст:
---
[текст]
---
```

**Аудит живого текста (детектор AI)**
```
Используй паттерны из writing-rules.md как чек-лист.
Дополнительно: методика и шкала из audit.md.

Тип текста: [пост / статья / письмо / лендинг]

Текст:
---
[текст]
---
```

**Аудит документации**
```
Используй writing-rules.md и .claude/skills/write-docs/SKILL.md как правила.
Методику и шкалу 1–10 бери из .claude/skills/audit-docs/SKILL.md.

Тип страницы: [справочная статья / описание метода / страница параметра / обзорная]

Текст:
---
[текст]
---
```
