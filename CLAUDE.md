# CLAUDE.md

TextSkill — набор правил и скиллов, которые заставляют тексты на русском звучать по-человечески, а не как генерация AI. Это не код: ядро правил + автозапускаемые скиллы Claude Code.

## Единый источник правды

- **`writing-rules.md`** — единственный источник паттернов: запрещённая лексика, синтаксис, структура, типографика, ритм, позитивные правила. Любое новое правило про живость текста идёт сюда.
- **`audit.md`** — только методика аудита (карта категорий + шкала + формат отчёта). **Не дублируй сюда стоп-листы.** Паттерны он берёт из ядра. Если переписываешь промт для обычного чата — ссылайся на ядро, а не копируй его.
- Форматные правила (структура поста, статьи, README) живут в соответствующем `.claude/skills/*/SKILL.md`, не в ядре.

## Маршрутизация скиллов

Скиллы срабатывают автоматически по описанию во фронтматтере. Логика выбора:

- «напиши пост / для тг» → **write-post** (короткий текст, до ~5 абзацев).
- «напиши статью / лонгрид / эссе» → **write-article** (от нескольких сотен слов).
- «напиши / обнови readme» → **write-readme**.
- «проверь / оцени текст, насколько живой» → **audit-text**.
- «перепиши / оживи / убери AI» → **rewrite-text**.

Граница audit ↔ rewrite: «оцени, что не так» — это audit (диагноз, балл). «Сделай лучше» — это rewrite (новый текст). Если из запроса неясно — спроси, что нужно: разбор или переписать.

Каждый скилл **сначала читает `writing-rules.md`**, потом применяет свои форматные правила. `audit-text` и `rewrite-text` дополнительно подтягивают форматный скилл под тип текста: пост → write-post, статья → write-article, README → write-readme.

## Чего пока нет

Генераторов под **письмо, продающий лендинг, описание продукта** нет. `audit`/`rewrite` знают эти типы, но написать с нуля такой текст профильным скиллом нельзя — запрос уйдёт в общий чат. Если нужен такой формат — это новый скилл, а не правка существующего.

## Дисциплина правок

- **Не раздувай списки.** Стоп-листы уже покрывают основные каталоги AI-маркеров (Википедия RU/EN, инфостиль). Ещё одна строка почти не двигает качество, но грузит контекст. Лучше править формулировки в ядре, чем добавлять пункты.
- **Держи примеры краткими.** В скиллах с примерами (write-post, audit-text, rewrite-text) образец якорит планку — но один и компактный.
- **Синхронизация не нужна** — раз паттерны в одном месте, дублей нет. Если заметил копию стоп-листа где-то ещё — это баг, убери.

## Допущение путей

Скиллы читают `writing-rules.md` и `audit.md` из корня проекта. Копируешь скиллы в другой репозиторий — тащи ядро вместе с ними, иначе скиллы сломаются.
